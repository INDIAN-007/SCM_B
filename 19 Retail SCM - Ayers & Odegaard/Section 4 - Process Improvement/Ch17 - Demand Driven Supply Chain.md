## Vision: From Forecast-Driven to Demand-Driven

**Forecast-driven SC:** Build to a plan; push inventory into the chain based on projected demand.
**Demand-driven SC:** Pull inventory through the chain based on actual consumption.

| Dimension | Forecast-Driven | Demand-Driven |
|---|---|---|
| Trigger | Forecast (prediction) | Actual customer demand (signal) |
| Inventory location | Pushed into stores and DCs | Pulled by actual replenishment signals |
| Bullwhip effect | High | Low (demand signal is clean) |
| Responsiveness | Slow (lag between signal and action) | Fast (replenishment based on real data) |
| Risk | Over/understock due to forecast error | Lower risk; responds to reality |

---

## The 3C Model (Alternative to MRP II)

Ayers & Odegaard propose the **3C Model** as a simpler alternative to complex MRP-based demand-driven replenishment:

| C | Meaning | Description |
|---|---|---|
| **Connect** | Connect supply chain nodes | Real-time data linkage between POS, DC, supplier |
| **Collaborate** | Collaborate on planning | Shared forecasts, joint S&OP, CPFR processes |
| **Coordinate** | Coordinate replenishment | Synchronised ordering cycles; Kanban-style triggers |

## Product Types in the Demand-Driven SC

| Product Type | Demand Signal | Replenishment Approach |
|---|---|---|
| **Functional / Staple** | POS consumption signal | Continuous replenishment; Kanban or ROP |
| **Innovative / Fashion** | Early sales trend signal | Test-and-repeat; OTB; no automatic replenishment |

## Barriers to the Demand-Driven SC

| Barrier | Description | Solution |
|---|---|---|
| Forecast dependence culture | "We always use a forecast" | Pilot demand-driven for one category; show results |
| Technology gaps | No real-time POS-to-DC-to-supplier visibility | Invest in EDI or API integration |
| Supplier resistance | Suppliers want advance orders; resist pull model | Share benefits; offer longer-term relationship |
| Organisational silos | Buying, logistics, suppliers don't communicate | CPFR; S&OP; cross-functional teams |
| SKU complexity | Too many SKUs to manage demand-driven | ABC analysis; focus on A items first |

## Demand-Driven Tools and Techniques

### Operating Improvements

| Tool | Description |
|---|---|
| **Lean SC approaches** | Remove waste; reduce batch sizes; compress lead times |
| **Constraint management (TOC)** | Identify and exploit the bottleneck in the chain |
| **Quality improvements** | Reduce defects that trigger excess safety stock |
| **Design for commonality** | Use common components across products; reduce SC complexity |

### Management Improvements

| Tool | Description |
|---|---|
| **Synchronisation and fixed-interval planning** | All parties order/plan on the same cycle (e.g., every Monday) |
| **Simplification** | Reduce SKU count; reduce supplier count; reduce process complexity |

---

## Lean Supply Chain in Demand-Driven Context

Lean principles applied to the demand-driven retail SC:

| Lean Principle | SC Application |
|---|---|
| **Eliminate waste** | Remove non-value-adding inventory; eliminate excess handling |
| **One-piece flow** | Replenish in smaller, more frequent quantities |
| **Pull system** | Replenishment triggered by consumption, not forecast |
| **Takt time** | Match replenishment rate to consumption rate |
| **Continuous improvement** | PDCA/DMAIC cycles on replenishment performance |
