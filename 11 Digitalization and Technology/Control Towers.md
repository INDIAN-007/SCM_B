# Supply Chain Control Towers

> **Source:** Kuldeepak Ch.5
> **Tags:** #technology #controltower #visibility

---

## Definition

A Supply Chain Control Tower is a centralised hub that provides **end-to-end visibility, analytics, and decision support** across the entire supply chain network.

Popularised by Amazon, Flipkart, and other e-commerce leaders.

---

## Three Functions (Kuldeepak)

### 1. Visibility
Four layers of supply chain visibility:
1. **Data Feed** — deploy digital means for easy data collection (IoT, EDI, API, manual)
2. **Data Consolidation** — automated consolidation via software (ERP, WMS, TMS)
3. **Dashboards + Mobility** — secure, role-based dashboards accessible on mobile
4. **Alerts on Exceptions** — real-time alerts when deviations occur; corrective actions triggered

### 2. Data Analysis and Forecasting
| Level | Description |
|---|---|
| **Descriptive** | What happened? Pre-defined metrics processed automatically |
| **Predictive** | What will happen? Statistical models and scenario planning |
| **Prescriptive** | What should we do? AI and big data recommend actions |

### 3. Collaboration and Integrated Optimization
| Type | Description |
|---|---|
| **Horizontal** | Between suppliers and customers |
| **Vertical** | Between supply chain partners across tiers |
| **Network (Co-opetition)** | All stakeholders including competitors collaborate on shared infrastructure |

**Co-opetition:** Cooperation with competition to leverage each other's strength and create overall SC efficiency.

---

## Implementation Framework (Kuldeepak Tip)

Start with:
1. **Define what to Track, Measure, Control** (use KPI framework)
2. **Decide granularity** — route level, SKU level, supplier level
3. **Deploy digital tools** to capture the data
4. **Build dashboards** — move from visibility to predictive to prescriptive
